Завдання проекту

Загальні завдання та налаштування проекту

Цей розділ описує базові вимоги до налаштування проекту та загальні аспекти розробки, які забезпечать його функціональність та адаптивність на різних пристроях.

  • Налаштувати базову структуру React-проекту.
  • Реалізувати адаптивну та гумову верстку з урахуванням брейкпоінтів:
    • Mobile: від 320px (гумова), з 375px (адаптивна).
    • Tablet: з 768px.
    • Desktop: з 1440px.
  • Забезпечити семантичну верстку.
  • Підключити шрифти.
  • Оптимізувати розміри статичної графіки (векторної та растрової).
  • Забезпечити підтримку відображення статичних зображень для Retina-екранів.
  • Оптимізувати завантаження зображень.
  • Підключити всі SVG-іконки через спрайт.
  • Додати відображення фавікон сторінки.

UI Kit / SharedLayout (Загальні компоненти)

Цей розділ деталізує розробку основних компонентів інтерфейсу користувача, які будуть повторно використовуватися по всьому додатку для забезпечення єдиного стилю та функціональності.

SharedLayout

  • Створити компонент `SharedLayout`, який рендериться на маршруті `/`.
  • Інтегрувати `Header` та `Footer` у `SharedLayout`.
  • Налаштувати `SharedLayout` для обгортання вкладених маршрутів та сторінок.

Header

  • Створити компонент `Header`.
  • Інтегрувати `Logo` у `Header`.
  • Реалізувати умовний рендеринг:
    • `Nav` (для авторизованого користувача).
    • `AuthBar` (для неавторизованого користувача).
    • `UserBar` (для авторизованого користувача).
  • Для авторизованого користувача:
    • Реалізувати навігаційний блок з маршрутами:
      • `/` (публічний, веде на `HomePage`).
      • `/add` (приватний, веде на `AddRecipePage`).
    • Реалізувати бургер-меню для мобільної версії, яке виїжджає справа і має розміри в'юпорту.

Nav

  • Створити компонент `Nav`.
  • Забезпечити клікабельність `Nav` для переадресації на `HomePage`.

Logo

  • Створити компонент `Logo` (для хедера та футера).
  • Забезпечити клікабельність `Logo` для переадресації на `HomePage`.

AuthBar

  • Створити компонент `AuthBar`.
  • Реалізувати 2 кнопки:
    • Sign in: по кліку відкривати `Modal` з `SignInModal`.
    • Sign up: по кліку відкривати `Modal` з `SignUpModal`.

UserBar

  • Створити компонент `UserBar`.
  • Відображати актуальну аватарку та ім'я користувача.
  • Реалізувати відображення дефолтного зображення при відсутності аватарки.
  • По кліку на компонент відображати випадаючий список:
    • Посилання Profile: перенаправляти на `UserPage` з інформацією про авторизованого користувача.
    • Кнопка Log out: по кліку відкривати `Modal` з `LogOutModal`.

Footer

  • Створити компонент `Footer`.
  • Інтегрувати `Logo` у `Footer`.
  • Інтегрувати `NetworkLinks` у `Footer`.
  • Інтегрувати `Copyright` у `Footer`.

NetworkLinks

  • Створити компонент `NetworkLinks`.
  • Реалізувати список посилань на соцмережі (`ul > li`):
    • Facebook: https://www.facebook.com/goITclub/
    • Instagram: https://www.instagram.com/goitclub/
    • YouTube: https://www.youtube.com/c/GoITNetworkLinks
  • Забезпечити відкриття посилань у новій вкладці.

Copyright

  • Створити компонент `Copyright`.
  • Відображати текст: "@2025, Foodies. All rights reserved".

Сторінка HomePage (публічний маршрут "/")

Цей розділ описує структуру та функціональність головної публічної сторінки додатку, яка слугує точкою входу для користувачів.

  • Створити сторінку `HomePage`.
  • Інтегрувати компоненти: `Hero`, `Categories`, `Recipes`, `Testimonials`.

Hero

  • Створити компонент `Hero`.
  • Інтегрувати `MainTitle` та `Subtitle`.
  • Додати кнопку "Add recipe":
    • Для неавторизованого користувача: по кліку відкривати `Modal` з `SignInModal`.
    • Для авторизованого користувача: по кліку перенаправляти на `AddRecipePage`.

MainTitle

  • Створити універсальний компонент `MainTitle`.

Subtitle

  • Створити універсальний компонент `Subtitle`.

Categories

  • Створити компонент `Categories`.
  • Інтегрувати `CategoryList`.

CategoryList

  • Створити компонент `CategoryList`.
  • Відображати список карток категорій рецептів, кожна з яких містить:
    • Зображення категорії.
    • Назва категорії.
    • Кнопку-іконку-стрілку:
      • По кліку відправляти запит на бекенд за рецептами обраної категорії.
      • Обробити помилку бекенду: відображати спливаюче повідомлення.
      • При успішному запиті: приховати `Categories` та відобразити `Recipes`.

Recipes

  • Створити компонент `Recipes`.
  • Інтегрувати: кнопку "Back", `MainTitle`, `Subtitle`, `RecipeFilters`, `RecipeList`, `RecipePagination`.
  • Кнопка "Back":
    • По кліку відображати `Categories` та приховувати `Recipes`.

RecipeFilters

  • Створити компонент `RecipeFilters`.
  • Реалізувати стилізований `select` для вибору інгредієнта:
    • По кліку відображати випадаючий список інгредієнтів з бекенду.
    • Кешувати список інгредієнтів у `Redux Store`.
  • Реалізувати стилізований `select` для вибору регіону:
    • По кліку відображати випадаючий список регіонів з бекенду.
    • Кешувати список регіонів у `Redux Store`.
  • При кожній зміні `select` скидати сторінку `RecipePagination` на 1.
  • Відправляти запит на бекенд з урахуванням всіх параметрів фільтрації та пагінації (за замовчуванням фільтри відсутні).

RecipeList

  • Створити компонент `RecipeList`.
  • Відображати список рецептів з використанням універсального компонента `RecipeCard`.

RecipeCard

  • Створити універсальний компонент `RecipeCard`.
  • Відображати:
    • Зображення страви.
    • Назва рецепту.
    • Опис рецепту.
    • Інформацію про автора (аватарка, ім'я) у вигляді кнопки:
      • Для неавторизованого користувача: по кліку відкривати `Modal` з `SignInModal`.
      • Для авторизованого користувача: по кліку перенаправляти на `UserPage` профілю автора.
    • Кнопку-іконку-серце (додати/видалити з обраного):
      • По кліку відправляти запит на бекенд.
      • Підсвічувати іконку, якщо рецепт в обраному.
    • Кнопку-іконку-стрілочку: по кліку перенаправляти на `RecipePage`.

RecipePagination

  • Реалізувати "серверную" пагінацію.
  • По кліку на порядковий номер відправляти запит на бекенд з урахуванням фільтрації та пагінації.
  • За замовчуванням порядковий номер сторінки = 1.

Testimonials

  • Створити компонент `Testimonials`.
  • Отримувати відгуки з бекенду.
  • Відображати:
    • Підзаголовок секції.
    • Заголовок секції.
    • Слайдер відгуків:
      • Кожен слайд містить текст відгуку та ім'я автора.
      • Автоматична прокрутка слайдів за заданим інтервалом.
      • Можливість ручної прокрутки за допомогою пагінації під слайдером.

Сторінка RecipePage (публічний маршрут "/recipe/:id")

Цей розділ описує деталізовану сторінку рецепту, доступну для публічного перегляду.

  • Створити сторінку `RecipePage`.
  • Інтегрувати: `PathInfo`, `RecipeInfo`, `PopularRecipes`.

PathInfo

  • Створити універсальний компонент `PathInfo`.
  • Відображати посилання на `HomePage` та назву поточної сторінки.

RecipeInfo

  • Створити компонент `RecipeInfo`.
  • До рендерингу отримати `id` рецепта з URL та відправити запит на бекенд.
  • Інтегрувати: `RecipeMainInfo`, `RecipeIngredients`, `RecipePreparation`.

RecipeMainInfo

  • Створити компонент `RecipeMainInfo`.
  • Приймати дані в пропсах та відображати:
    • Зображення рецепта.
    • Назва страви.
    • Категорія страви.
    • Опис страви.
    • Інформацію про автора (у вигляді кнопки):
      • Для неавторизованого користувача: по кліку відкривати `Modal` з `SignInModal`.
      • Для авторизованого користувача: по кліку перенаправляти на `UserPage` профілю автора.

RecipeIngredients

  • Створити компонент `RecipeIngredients`.
  • Приймати дані в пропсах та відображати список інгредієнтів, кожен з яких містить:
    • Зображення інгредієнта або плейсхолдер.
    • Назва інгредієнта.
    • Його кількість.

RecipePreparation

  • Створити компонент `RecipePreparation`.
  • Приймати дані в пропсах та відображати:
    • Опис приготування страви.
    • Кнопку "Add to favorites" / "Remove from favorites":
      • По кліку відправляти запит на бекенд для додавання/видалення рецепта з обраного.
      • Змінювати текст кнопки залежно від статусу.

PopularRecipes

  • Створити компонент `PopularRecipes`.
  • Відображати заголовок секції.
  • Відображати список популярних рецептів з використанням `RecipeCard`.

Сторінка AddRecipePage (приватний маршрут "/recipe/add")

Цей розділ описує сторінку для додавання нових рецептів, доступну лише авторизованим користувачам.

  • Створити сторінку `AddRecipePage`.
  • Інтегрувати: `PathInfo`, `MainTitle`, `Subtitle`, `AddRecipeForm`.

AddRecipeForm

  • Створити компонент `AddRecipeForm`.
  • Реалізувати форму з використанням `react-hook-form` та `Yup` для валідації.
  • Поля форми:
    • `input` типу `file` для фото рецепта (використовувати `URL.createObjectURL()` для тимчасового відображення).
    • `input` для назви рецепта.
    • `input` для короткого опису (до 200 символів) з лічильником.
    • Стилізований `select` для категорії рецепта (список з бекенду).
    • Стилізований лічильник часу приготування (не менше 1 хвилини).
    • Стилізований `select` для додавання інгредієнта (список з бекенду).
    • `input` для кількості вибраного інгредієнта.
    • Кнопка "Add ingredient+":
      • По кліку зберігати інформацію про інгредієнт у списку.
    • Список доданих інгредієнтів (картки із зображенням, назвою, кількістю та кнопкою видалення).
    • `textarea` для інструкції з приготування (до 200 символів) з лічильником.
    • Кнопка-іконка-кошик: по кліку очищати всі поля форми.
    • Кнопка `submit` "Publish":
      • Активувати валідацію (всі поля обов'язкові).
      • Підсвічувати невалідні поля червоним та інформувати про причину.
      • При валідних даних відправляти запит на бекенд (`FormData`).
      • Обробити помилку бекенду: відображати спливаюче повідомлення.
      • При успішному запиті: перенаправляти користувача на `UserPage`.

Сторінка UserPage (приватний маршрут "/user/:id")

Цей розділ описує сторінку профілю користувача, яка може відображати власну інформацію або інформацію про інших користувачів.

  • Створити сторінку `UserPage`.
  • Інтегрувати: `PathInfo`, `MainTitle`, `Subtitle`, `UserInfo`, кнопку "Log Out", `TabsList`, `ListItems`, `ListPagination`.
  • Кнопка "Log Out":
    • По кліку відкривати `Modal` з `LogOutModal`.
  • Кнопка "Follow" / "Following" (для профілю іншого користувача):
    • "Follow": по кліку відправляти запит на бекенд для підписки, потім змінювати на "Following".
    • "Following": по кліку відправляти запит на бекенд для відписки, потім змінювати на "Follow".

UserInfo

  • Створити компонент `UserInfo`.
  • Відображати аватарку користувача (дефолтне зображення при відсутності).
  • `input` типу `file` для додавання аватарки (тільки на власному профілі).
  • Відображати додаткову інформацію про користувача: email, кількість створених рецептів, кількість обраних рецептів, кількість підписників, кількість підписок.

TabsList

  • Створити компонент `TabsList`.
  • Реалізувати вкладки:
    • Для власного профілю:
      • "My recipes" (створені рецепти).
      • "My favorites" (обрані рецепти).
      • "Followers" (список підписників).
      • "Following" (список підписок).
    • Для чужого профілю:
      • "Recipes" (рецепти власника).
      • "Followers" (список підписників власника).
  • По кліку на вкладку скидати сторінку `ListPagination` на 1.
  • Відправляти відповідний запит на бекенд з урахуванням пагінації.
  • Обробити помилку бекенду: відображати дефолтний контент у `ListItems`.
  • При успішному запиті: `ListItems` відображає отриману інформацію.
  • Підсвічувати активну вкладку.

ListItems

  • Створити компонент `ListItems`.
  • Приймати дані в пропсах та відображати список `RecipePreview` або `UserCard` залежно від вибраної вкладки.

RecipePreview

  • Створити компонент `RecipePreview`.
  • Відображати:
    • Зображення рецепта.
    • Назва рецепта.
    • Короткий опис.
    • Посилання на `RecipePage` (іконка-стрілочка).
    • Кнопка-іконка-кошик:
      • По кліку відправляти запит на бекенд для видалення рецепта (зі своїх або обраних).
      • Видаляти картку без перезавантаження сторінки.

UserCard

  • Створити компонент `UserCard`.
  • Відображати:
    • Аватарку користувача.
    • Ім'я користувача.
    • Кількість власних рецептів.
    • Список рецептів користувача (для планшетів/десктопів).
    • Посилання на `UserPage`.
    • Кнопка:
      • Для вкладки "Followers":
        • "Follow": по кліку відправляти запит на бекенд для підписки, потім змінювати на "Following".
        • "Following": по кліку відправляти запит на бекенд для відписки, потім змінювати на "Follow".
      • Для вкладки "Following":
        • "Following": по кліку відправляти запит на бекенд для відписки.
        • Видаляти картку користувача зі списку без перезавантаження сторінки.

ListPagination

  • Реалізувати "серверную" пагінацію.
  • По кліку на порядковий номер відправляти запит на бекенд залежно від вкладки, передаючи номер сторінки.
  • При видаленні останнього елемента на сторінці перенаправляти на попередню сторінку.

Модальні вікна (Modals)

Цей розділ описує реалізацію модальних вікон, які використовуються для взаємодії з користувачем.

Modal

  • Створити універсальний компонент `Modal`.
  • Відображає контент, переданий як `children`.
  • Містить 1 функціональну кнопку для закриття.
  • Закривається по: кліку на кнопку-іконку закриття, кліку по бекдропу, натисканню `Escape`.

SignUpModal

  • Створити компонент `SignUpModal`.
  • Відображати заголовок.
  • Інтегрувати `SignUpForm`.
  • Кнопка "Sign in": по кліку відображати `SignInModal`.

SignUpForm

  • Створити компонент `SignUpForm`.
  • Форма з `input`'ами: Name, Email, Password.
  • Кнопка `submit` "Create".
  • `Input Password`:
    • Кнопка-іконка-око (закреслене/відкрите).
    • По кліку на закреслене око: змінювати іконку на відкрите око, робити пароль видимим.
    • По кліку на відкрите око: змінювати іконку на закреслене око, приховувати пароль (зірочки).
  • Валідація по `submit` з використанням `react-hook-form` та `Yup`.
  • При невалідних значеннях: відображати причину помилки, не відправляти дані на бекенд.
  • При валідних значеннях: відправляти дані на бекенд.
  • Обробити помилку бекенду: відображати спливаюче повідомлення.
  • При успішному запиті (сформовано `token`): реалізувати автоматичну авторизацію, закривати `Modal`.

SignInModal

  • Створити компонент `SignInModal`.
  • Відображати заголовок.
  • Інтегрувати `SignInForm`.
  • Кнопка "Create an account": по кліку відображати `SignUpModal`.

SignInForm

  • Створити компонент `SignInForm`.
  • Форма з `input`'ами: Email, Password.
  • Кнопка `submit` "Sign in".
  • Валідація по `submit` з використанням `react-hook-form` та `Yup`.
  • При невалідних значеннях: відображати причину помилки, не відправляти дані на бекенд.
  • При валідних значеннях: відправляти дані на бекенд.
  • Обробити помилку бекенду: відображати спливаюче повідомлення.
  • При успішному запиті (сформовано `token`): закривати `Modal`.

LogOutModal

  • Створити компонент `LogOutModal`.
  • Відображати питання.
  • Кнопка "Cancel": по кліку закривати `Modal`.

    • По кліку відправляти запит на бекенд для видалення активної сесії.
    • Незалежно від відповіді бекенду: деавторизувати "на клієнті" (очистити `Redux Store` та `localStorage`).
    • Перенаправити користувача на `HomePage`.
